# 🚫 ЧТО НЕ ОТПРАВЛЯЕТСЯ НА GITHUB И ПОЧЕМУ

## ✅ КОРОТКИЙ ОТВЕТ

**На GitHub отправляются ТОЛЬКО файлы с кодом:**
- `.cpp`, `.h` файлы (C++)
- `.py` файлы (Python)
- `CMakeLists.txt`, `requirements.txt`
- `README.md`, документация
- `.gitignore`, конфиги

**НЕ отправляются временные и системные файлы** (см. таблицу ниже).

---

## 🔍 КАК ПРОВЕРИТЬ ЧТО БУДЕТ ОТПРАВЛЕНО?

### Перед коммитом

```bash
# Посмотреть какие файлы Git видит
cd C:\MyProjects
git status

# Показывает только ОТСЛЕЖИВАЕМЫЕ файлы
# Всё что в .gitignore - НЕ показывается!
```

### Полный список отслеживаемых файлов

```bash
git ls-files
# Выведет ВСЕ файлы которые будут на GitHub
```

### Проверить один проект

```bash
# Только C++
git ls-files C++/

# Только Python
git ls-files Python/
```

## 📝 ПРИМЕРЫ

### Пример 1: Добавили новый C++ файл

```bash
# Вы создали: C++/src/Calculator.cpp
# Вы создали: C++/include/Calculator.h

git status
# Показывает: C++/src/Calculator.cpp (NEW)
#             C++/include/Calculator.h (NEW)

git add C++/
git commit -m "feat(cpp): add Calculator"
git push

# На GitHub появятся Calculator.cpp и Calculator.h ✅
# build/ папка НЕ появится (в .gitignore)
```

### Пример 2: Запустили программу

```bash
# Вы запустили и создались файлы:
# C++/build/CMakeFiles/...    ← Генерируется CMake
# C++/build/bin/MyProjects.exe ← Скомпилировано
# Python/__pycache__/main.cpython-39.pyc ← Кэш Python

git status
# Показывает: (nothing to commit) - ВСЕ в .gitignore!

# На GitHub ничего нового не появится ✅
```

### Пример 3: Установили пакет Python

```bash
# Вы установили: pip install numpy
# Создалась папка: Python/venv/lib/site-packages/numpy/

git status
# Показывает: requirements.txt (MODIFIED) - только этот!

pip freeze > Python/requirements.txt

git add Python/requirements.txt
git commit -m "feat(python): add numpy"
git push

# На GitHub: requirements.txt обновился ✅
# На GitHub: venv/ папка НЕ загружена ✅
```

## 🆘 ЕСЛИ СЛУЧАЙНО ЗАКОММИТИЛИ ЛИШНЕЕ

### Удалить из Git (но оставить на диске)

```bash
# Удалили build/ из Git
git rm -r --cached C++/build/

# Коммитьте удаление
git commit -m "remove build from tracking"
git push
```

### Проверить что удалилось

```bash
git ls-files
# build/ больше НЕ будет в списке!
```

## 🎓 ГЛАВНОЕ ПРАВИЛО

> **.gitignore защищает от коммита лишних файлов автоматически** ✅

**Вы можете смело:**
- Собирать проект (build/ создастся, но не закоммитится)
- Устанавливать пакеты (venv/ создастся, но не закоммитится)
- Редактировать в IDE (временные файлы создадутся, но не закоммитятся)

**Всё работает автоматически!** 🚀

## ✨ ИТОГ

На GitHub будет **ТОЛЬКО**:
- ✅ Исходный код (.cpp, .h, .py)
- ✅ Конфигурация (CMakeLists.txt, requirements.txt)
- ✅ Документация (README.md, гайды)
- ✅ .gitignore файл

**НЕ будет:**
- ❌ Временные папки (build/, venv/, __pycache__)
- ❌ Скомпилированные файлы (*.exe, *.o, *.pyc)
- ❌ Личные настройки (.idea/)
- ❌ Кэш и логи

**Всё контролируется .gitignore автоматически!** 🎉
